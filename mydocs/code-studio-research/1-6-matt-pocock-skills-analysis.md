## 核心洞察（TL;DR）

1. **Skills 应该小而美**：每个 skill 只做一件事，易于理解、修改、组合
2. **边做边记录**：决策 crystallize 时立即更新文档，避免"文档腐烂"
3. **领域语言是关键**：`CONTEXT.md` 是 Matt Pocock Skills 的杀手级特性
4. **Feedback Loop 是第一位的**：`diagnose` skill 的核心是"先构建 feedback loop"
5. **不要过度框架化**：提供工具，不拥有流程（与 CCGS 的区别）
