## 步骤1：环境准备（预计30分钟）

### 1.1 克隆仓库
```bash
cd ~/VackyAI  # 或你喜欢的目录
git clone git@github.com:VackyZhang/hermes-agent.git
cd hermes-agent
git checkout vacky/dev  # 确保使用你的开发分支
```

### 1.2 创建虚拟环境
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 1.3 安装依赖
```bash
pip install -e ".[dev]"
```

### 1.4 配置 API Key
```bash
# 创建配置目录和文件
mkdir -p ~/.hermes
cat > ~/.hermes/.env << EOF
# 选择一个可用的 provider（根据 handbook 中的推荐模型等级）
# 选项1：Anthropic（推荐 Claude 3.7 Sonnet）
ANTHROPIC_API_KEY=your_actual_key_here

# 选项2：OpenAI
# OPENAI_API_KEY=your_actual_key_here

# 选项3：Google（免费套餐可用）
# GEMINI_API_KEY=your_actual_key_here
EOF
```

### 1.5 验证安装
```bash
hermes --version
# 应该输出类似：hermes-agent x.x.x
```

**✅ 完成标准**：
- [ ] `hermes --version` 成功输出版本号
- [ ] `~/.hermes/.env` 文件已创建并包含有效的 API Key

## 附录：常见问题

### Q1：API Key 配置失败怎么办？
- 检查 `~/.hermes/.env` 文件是否存在
- 检查 API Key 是否有效（尝试在官网控制台验证）
- 检查环境变量是否被正确加载（可以尝试 `echo $ANTHROPIC_API_KEY`）

### Q2：Hermes Agent 启动失败怎么办？
- 检查虚拟环境是否激活（`which python` 应该指向 `.venv/bin/python`）
- 检查依赖是否安装完整（`pip list | grep hermes`）
- 查看错误日志（`hermes --verbose` 或查看 `~/.hermes/logs/agent.log`）

### Q3：交互时遇到奇怪的回复怎么办？
- 这是正常的，记录下这个观察
- 尝试重新提问，看是否能得到更好的回复
- 如果问题持续，可以尝试 `/reset` 重置对话

### Q4：如何判断"意外观察"是否有效？
- 有效的观察应该是具体的、可验证的
- 例如："工具调用时响应变慢" 是有效观察
- 例如："它很聪明" 是无效观察（太主观、不可验证）
